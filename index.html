üìÅ 2. File Systems & Management

Core File System Services:

  1. Create files
  2. Open/Close files
  3. Read/Write data
  4. Delete files
  5. Manage metadata (size, date, permissions)

Uniform I/O Interface:

  • Everything is a file in UNIX
  • Same commands work for files, devices, pipes
  • cat file.txt and cat /dev/mouse use same interface

üí° Memory Aid:

Think of file system like a library:

  • Files = Books
  • Directories = Shelves/Sections
  • Permissions = Library card levels
  • Metadata = Book catalog information

üß† 3. Memory Management

Key Problems to Solve:

  1. Multiple processes sharing memory
  2. Protection between processes
  3. Performance optimization
  4. Memory allocation efficiency

Address Types:

  • Logical Address: What the program sees
  • Physical Address: Actual memory location
  • Virtual Address: Same as logical (in virtual memory systems)

Memory Management Unit (MMU):

CPU ‚Üí [Logical Address] ‚Üí MMU ‚Üí [Physical Address] ‚Üí Memory

üí° Memory Aid:

Think of memory like parking spaces:

  • Fixed partitions = Assigned parking spots
  • Dynamic allocation = First-come, first-served parking
  • Fragmentation = Empty spaces too small for cars

üíæ 4. Virtual Memory

What is Virtual Memory?

  • Illusion of large memory using disk storage
  • Programs can be larger than physical RAM
  • Each process has its own virtual address space

Page Address Translation:

üîÑ 6. CPU Scheduling

Multitasking Types:

  1. Cooperative: Processes voluntarily give up CPU
  2. Preemptive: OS forces process switches

Scheduling Goals:

  • Low latency (quick response)
  • High throughput (more work done)
  • Fairness (all processes get turns)

üí° Memory Aid:

Think of CPU scheduling like a traffic light system:

  • Timeslice = Green light duration
  • Context switch = Traffic light change
  • I/O-bound processes = Pedestrians (quick crossing)
  • CPU-bound processes = Trucks (need longer green light)

üîó 7. Inter-Process Communication (IPC)

Why Do Processes Need to Communicate?

  • Share data
  • Coordinate activities
  • Notify about events

IPC Mechanisms:

  • Pipes: Data flows one direction (FIFO)
  • Semaphores: Control access to resources
  • Message Queues: Send/receive messages
  • Shared Memory: Direct memory sharing (fastest)

üí° Memory Aid:

Think of IPC like communication methods:

  • Pipes = Telephone line
  • Shared Memory = Whiteboard in shared room
  • Message Queues = Email system
  • Signals = Text messages

üì° 8. Signals

What are Signals?

  • Software interrupts
  • Asynchronous notifications
  • Cause predefined actions or custom handlers

Common UNIX Signals:

  • SIGINT (2): Interrupt (Ctrl+C)
  • SIGTERM (15): Termination request
  • SIGKILL (9): Force kill (cannot be caught)
  • SIGALRM (14): Timer alarm
  • SIGHUP (1): Hangup

üí° Memory Aid:

Think of signals like emergency alerts:

  • SIGINT = Fire alarm (can be disabled)
  • SIGKILL = Sprinkler system (cannot be disabled)
  • SIGTERM = Polite evacuation request
  • Signal handlers = Emergency response plans

üîí 9. Synchronization & Deadlocks

Critical Section Problem:

  • Code section accessing shared resources
  • Only one process should be in critical section at a time
  • Need mutual exclusion

Four Deadlock Conditions (ALL must be present):

  1. Mutual Exclusion: Resources cannot be shared
  2. Hold and Wait: Process holds resources while waiting for more
  3. No Preemption: Resources cannot be forcibly taken
  4. Circular Wait: Circular chain of waiting processes

üí° Memory Aid:

Think of deadlock like traffic gridlock:

  • Mutual exclusion = Only one car per lane
  • Hold and wait = Cars blocking intersections
  • No preemption = Cannot move cars out of way
  • Circular wait = Cars waiting in circle

üíø 10. I/O & Device Management

Device Types:

  • Block devices: Disks (fixed-size blocks)
  • Character devices: Keyboards, mice (character streams)
  • Network devices: Network interfaces

Device Drivers:

  • Software that controls hardware
  • Translate generic I/O requests to device-specific operations
  • Hide complexity from applications

üí° Memory Aid:

Think of device drivers like translators:

  • Applications speak "English"
  • Devices speak different "languages"
  • Drivers translate between them
  • Generic I/O provides common "dictionary"

üîß 11. System Calls

What are System Calls?

  • Interface between user programs and kernel
  • Controlled way to access OS services
  • Require mode switch (user ‚Üí kernel)

Common System Calls:

  • File operations: open(), read(), write(), close()
  • Process control: fork(), exec(), wait(), exit()
  • Memory management: brk(), mmap()
  • Communication: pipe(), socket()

üí° Memory Aid:

Think of system calls like going to government office:

  • Library functions = Things you can do at home
  • System calls = Things requiring official permission
  • Mode switch = Security checkpoint to enter building
  • Kernel = Government officials with special powers

üìà 12. Performance & Optimization

Key Performance Concepts:

  • Thrashing: Excessive page swapping
  • Locality of Reference: Programs access nearby/recent memory
  • Working Set: Pages process actively uses

Performance Monitoring:

  • ps command: Shows currently running processes
  • top command: Real-time system performance
  • Swappiness (0-100): Controls swap aggressiveness

üí° Memory Aid:

Think of performance like traffic flow:

  • Thrashing = Traffic jam (cars moving very slowly)
  • Locality = Most trips are to nearby places
  • Working set = Places you visit regularly
  • Monitoring tools = Traffic cameras and reports

üõ°Ô∏è 13. Security

Security Principles:

  • Principle of Least Privilege: Give minimum access needed
  • Memory Protection: Base and limit registers check bounds
  • Access Control: Authentication, Authorization, Auditing

Common Security Threats:

  • Buffer Overflow: Writing past buffer boundaries
  • Privilege Escalation: Gaining unauthorized access
  • Race Conditions: Timing-dependent vulnerabilities

üí° Memory Aid:

Think of security like building security:

  • Least privilege = Key cards with limited access
  • Memory protection = Locked doors between departments
  • Buffer overflow = Climbing through windows
  • Access control = Security guards and cameras

üöÄ 14. Modern OS Features

Virtualization:

  • Traditional Virtualization: Hypervisor manages multiple OS instances
  • Containerization: OS-level virtualization, shared kernel

Kernel Architectures:

  • Monolithic Kernel: All services in kernel space
  • Microkernel: Minimal kernel, services in user space

üí° Memory Aid:

Think of modern features like building evolution:

  • Monolithic = Single large building
  • Microkernel = Campus with specialized buildings
  • Containers = Apartments in same building
  • VMs = Separate houses
